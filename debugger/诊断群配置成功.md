## 💡 使用技巧

### 技巧 1：专群专用

将不同类型的问题分配到不同的群：
- 诊断群 → 系统问题、Bug 排查
- 工作群 → 代码开发、日常任务
- 私聊 → 个人助理

### 技巧 2：快速切换（如果需要）

如果在诊断群想临时用 main：
```
/agent main
（临时使用 main）
/agent debugger  ← 切回 debugger
```

### 技巧 3：多个专用群

可以创建多个诊断群，都绑定到 debugger：
```json
{
  "bindings": [
    {
      "agentId": "debugger",
      "match": {
        "channel": "feishu",
        "peer": { "kind": "group", "id": "oc_诊断群1" }
      }
    },
    {
      "agentId": "debugger",
      "match": {
        "channel": "feishu",
        "peer": { "kind": "group", "id": "oc_诊断群2" }
      }
    }
  ]
}
```

## 🆘 如果遇到问题

### 问题 1：诊断群还是 main 回复

**排查**：
1. 确认 gateway 已重启
2. 运行 `openclaw agents list --bindings` 查看规则
3. 检查群组 ID 是否正确

**解决**：重启 gateway
```bash
systemctl --user restart openclaw-gateway
```

### 问题 2：想删除这个绑定

编辑配置文件，删除 bindings 中的对应条目，然后重启：
```bash
# 编辑配置
nano ~/.openclaw/openclaw.json

# 删除 bindings 部分或特定条目

# 重启
systemctl --user restart openclaw-gateway
```

### 问题 3：想添加更多群

1. 在飞书创建新群，拉入机器人
2. 在群里发消息
3. 查看日志获取群 ID：
   ```bash
   journalctl --user -u openclaw-gateway -n 100 --no-pager | grep "chat_id"
   ```
4. 告诉我新的群 ID，我帮您添加配置
