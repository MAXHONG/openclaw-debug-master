## ⚠️ 第二部分：不会自动调用（设计如此）

### 为什么不自动切换？

**1. 用户控制**
- Agent 切换需要用户明确同意
- 用户应该知道当前在和谁对话

**2. 会话隔离**
- Main Agent 和 Debug Master 有独立的对话历史
- 自动切换会丢失上下文

**3. 透明性**
- 避免"黑箱操作"
- 用户清楚知道哪个 Agent 在处理问题

### 当前行为（智能建议）

```
┌─────────────────────────────────────────┐
│  用户: "我的服务启动失败"                │
├─────────────────────────────────────────┤
│  Main Agent 识别到诊断关键词              │
│  ↓                                       │
│  判断: 这不是我的专长                     │
│  ↓                                       │
│  建议: "推荐切换到 Debug Master"          │
│  提供: 切换命令 + 诊断群链接              │
│  ↓                                       │
│  等待用户决定                             │
└─────────────────────────────────────────┘
```

## ✅ 第三部分：如何实现"自动协作"

虽然不会自动切换，但您可以通过以下方式实现自动化：

### 方案 1: 使用飞书诊断群（推荐，零操作）

```
┌──────────────┐           ┌──────────────┐
│  普通群/私聊  │           │   诊断群      │
│              │           │              │
│ Main Agent   │           │ Debug Master │
│ (自动路由)    │           │ (自动路由)    │
└──────────────┘           └──────────────┘
       ↓                          ↓
 "写个代码"               "服务启动失败"
       ↓                          ↓
  Main 处理               Debugger 处理
```

**优势**:
- ✅ 完全自动，无需手动切换
- ✅ 用户体验流畅
- ✅ 上下文不丢失（每个群独立）

---

### 方案 2: 共享诊断技能（半自动）

**实现步骤**:
```bash
# 将 Debug Master 的技能链接到 Main Agent
cd ~/.openclaw/workspace/skills
ln -s ../agents/debugger/skills/health-check.sh ./
ln -s ../agents/debugger/skills/service-check.sh ./
ln -s ../agents/debugger/skills/port-check.sh ./
```

**效果**:
```
用户: "检查系统健康"
Main Agent: [直接运行 health-check.sh]
            [返回诊断结果]
```

**优势**:
- ✅ Main Agent 也能执行诊断
- ✅ 无需切换 Agent

**劣势**:
- ⚠️ Main Agent 缺少诊断上下文（不如 Debug Master 专业）
- ⚠️ 没有自主学习能力

---

### 方案 3: 使用 Message Tool（高级）

在主 Agent 系统提示词中添加：

```markdown
### 自动委派诊断请求

当用户描述系统问题时，使用 message tool 向诊断群发送：

message.send({
  channel: 'feishu',
  peer: 'group:oc_a83975659a474e0b6d347a12c188085a',
  content: '@Debug Master 请帮用户诊断：[问题描述]'
})

然后告诉用户："我已将您的问题发送到诊断群，Debug Master 会在那里为您排查。"
```

**效果**:
```
用户在私聊: "我的服务挂了"
↓
Main Agent: [识别诊断问题]
            [自动向诊断群发消息]
            [告知用户去诊断群查看]
↓
用户去诊断群
↓
Debug Master 在诊断群回复
```

**优势**:
- ✅ 半自动化（Main Agent 帮忙转发）
- ✅ 保持专业分工
- ✅ 用户知道去哪里查看结果
