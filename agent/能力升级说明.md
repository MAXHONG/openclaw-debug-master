# Debug Master 能力升级说明

## 🎉 新增能力（2026-03-13）

### 1. 📸 图片分析能力 ⭐ 重要

Debug Master 现在可以分析您发送的截图！

**使用方式**：
- 直接在诊断群发送截图
- Debug Master 会自动使用 `gsk analyze` 分析
- 给出错误信息、原因和解决方案

**示例场景**：
```
你：（发送错误截图）
Debugger：
  分析截图内容...
  
  检测到错误：
  - 错误类型：502 Bad Gateway
  - 原因：Nginx 上游服务器无响应
  - 解决方案：
    1. 检查后端服务是否运行
    2. 检查 Nginx 配置中的 proxy_pass
    3. 查看 Nginx 错误日志
```

**重要**：
- ✅ gsk analyze 是**完全免费的**
- ✅ 支持错误截图、配置截图、日志截图
- ✅ 可以分析本地文件和 URL

### 2. 🌐 远程服务器诊断能力

Debug Master 现在可以 SSH 连接到远程服务器进行诊断！

**使用方式**：
```
你：帮我诊断远程服务器 root@192.168.1.100
Debugger：
  好的，我会连接到服务器进行诊断。
  请确认您的 SSH 密钥已配置。
  
  （执行远程诊断）
  
  诊断结果：
  - 磁盘空间：90% 已使用（需要清理）
  - 内存：充足
  - 失败的服务：nginx.service
  - 建议：重启 nginx 服务
```

**支持的操作**：
- ✅ 远程执行命令
- ✅ 查看远程日志
- ✅ 检查远程服务状态
- ✅ 分析远程系统资源

**安全提示**：
- ⚠️ 需要事先配置 SSH 密钥
- ⚠️ Debug Master 会向您确认操作
- ⚠️ 敏感操作需要您授权

**快速诊断脚本**：
```bash
./skills/remote-diagnose.sh user@host [port]
```

### 4. 📊 系统化诊断流程

Debug Master 的诊断流程更加完善：

```
1. 明确问题
   └─ 询问症状、时间线、改动
   └─ 如果有截图，立即分析

2. 收集证据
   └─ 本地：systemctl, journalctl, ps, netstat
   └─ 远程：SSH 执行命令
   └─ 网络：gsk search 查找解决方案

3. 分析根因
   └─ 提取关键错误信息
   └─ 验证每个假设
   └─ 找到根本原因

4. 提出方案
   └─ 2-3 个可选方案
   └─ 推荐最佳方案
   └─ 详细操作步骤

5. 执行验证
   └─ 征得同意
   └─ 逐步执行
   └─ 验证结果
```

## 🔐 安全提示

### SSH 密钥配置

推荐使用 SSH 密钥而非密码：

```bash
# 生成密钥（如果没有）
ssh-keygen -t ed25519 -C "your-email@example.com"

# 复制公钥到远程服务器
ssh-copy-id user@remote-host

# 测试连接
ssh user@remote-host "echo OK"
```

### 权限控制

Debug Master 使用 `elevated` 权限：
- ✅ 可以执行 sudo 命令
- ⚠️ 会向您确认危险操作
- ✅ 操作前会说明影响

### 远程操作安全

- ✅ Debug Master 会显示要执行的命令
- ✅ 征得您同意后才执行
- ⚠️ 敏感信息（密码）不会记录
- ✅ 推荐使用密钥认证
